Important For Users

ถ้า Windows ขึ้นหน้าฟ้าเตือน ไม่ได้แปลว่าเครื่องเสีย

ตอนเปิดไฟล์ติดตั้ง Windows อาจขึ้นข้อความประมาณว่า “Windows protected your PC” เพราะแอปยังไม่ได้มีชื่อผู้เผยแพร่ที่ Windows รู้จักในวงกว้าง หน้าจอนี้เป็นการเตือนตามปกติของ Windows ไม่ใช่อาการไวรัสทำงานเองทันที

วิธีกดต่อแบบง่าย ๆ

  1. 1. เมื่อหน้าฟ้าเด้งขึ้นมา ให้มองหาปุ่ม More info
  2. 2. กด More info ก่อน
  3. 3. Windows จะโชว์รายละเอียดเพิ่ม และจะมีปุ่ม Run anyway
  4. 4. ถ้าคุณดาวน์โหลดไฟล์จากหน้านี้ ให้กด Run anyway เพื่อทำต่อ

คำอธิบายแบบภาษาคนทั่วไป

  • ถ้ายังไม่เห็นปุ่ม `Run anyway` ให้กด `More info` ก่อนเสมอ
  • หน้าต่างนี้เป็นการเตือนจาก Windows ว่าไฟล์ยังไม่เป็นที่รู้จัก ไม่ได้แปลว่าไฟล์นี้เปิดเองหรือทำลายเครื่องทันที
  • ถ้าโหลดจากแหล่งอื่นที่ไม่ใช่หน้านี้ หรือมีชื่อไฟล์แปลกไปจากที่ระบุไว้ ให้หยุดก่อนและอย่าเพิ่งกดต่อ
  • ไฟล์ที่ควรเปิดคือ `ReimbursementDocApp-Setup.exe` หลังแตก zip แล้ว

What It Is

offline Windows app

local workflow

สำหรับผู้ใช้ทั่วไป

  • ติดตั้งแล้วใช้ในเครื่องได้ทันที
  • กรอกข้อมูลครั้งเดียวแล้วออกเอกสารหลายใบ
  • เก็บ output ไว้ในโฟลเดอร์ของโปรแกรม
  • แก้ Word template ภายหลังได้ถ้าต้องการ

สำหรับคนที่อยากศึกษา

  • มี handoff อธิบายสถาปัตยกรรมและ business rules
  • แยก master data, preset data และ template registry ชัด
  • แนวคิด reusable docx engine สำหรับระบบเอกสารอื่น
  • มี workflow ใช้ AI ช่วย refine, debug, handoff และ iterate

Narrative ที่ต้องชัด

หน้านี้เป็น landing page สำหรับดาวน์โหลดโปรแกรมและศึกษาแนวคิดของระบบ แต่ตัวโปรแกรมจริงคือ WinForms app ที่รันในเครื่องผู้ใช้ ไม่ใช่ SaaS, ไม่ใช่ dashboard บนเว็บ

Architecture

แนวคิดระบบที่เอาไปใช้ซ้ำได้

Word template

ให้ layout อยู่ใน `.docx` แล้วใช้ code แทน `{tag}` แบบไม่ทำลายโครงหน้าเอกสาร

Master data JSON

เก็บข้อมูลอ้างอิงไว้ใน `app_database.json` เพื่อแยก business truth ออกจาก UI

User preset JSON

เก็บ preset ของผู้ใช้แยกจาก master data เพื่อรองรับงานรอบเดือนถัดไปแบบปลอดภัย

Offline packaging

ใช้ installer + payload + template folder เพื่อให้ workflow ใช้งานจริงในหน่วยงานได้ง่าย

AI Workflow

ใช้ AI ทำโปรแกรมเอกสารอย่างไรให้ต่อยอดได้จริง

01

ใช้ AI ช่วย refine requirement และแยกปัญหาเป็น UI, business logic, template, data, packaging

02

ให้ AI ช่วย debug เฉพาะจุด, สรุปผลกระทบ, และลดงานลองผิดลองถูกในส่วนที่เป็น code

03

ใช้ AI ช่วยเขียน handoff, สรุป source of truth, และรักษาบริบทให้ chat ถัดไปไม่เริ่มจากศูนย์

04

เก็บ business data ที่ยืนยันแล้วไว้ใน JSON ชัด ๆ ไม่ปล่อยให้ AI เดาแทนข้อมูลจริงของงานราชการ
